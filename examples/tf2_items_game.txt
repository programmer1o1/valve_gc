// Extended subset of TF2's items_game.txt for the tf2_gc schema parser.
// Def indexes, item slots, item levels, and particle IDs below are all
// pulled directly from a real TF2 install's tf/scripts/items/items_game.txt
// (2026-07-21) via a KeyValues parser + tf_english.txt localization lookup,
// not hand-typed -- the first version of this file used fabricated/
// hallucinated numbers that didn't exist in the real schema, which silently
// broke backpack display (see docs/tf2_live_hook.md), so every entry here is
// independently verified against the real schema rather than recalled from
// memory.
//
// Includes every non-cosmetic-skin weapon each class can use (stock +
// unlocks; War Paint/"Upgradeable" base items and per-skin War Paint variants
// excluded from the general weapon list -- those aren't distinct obtainable
// weapons, just reskin bases), a couple of Unusual-eligible hats, the Golden
// Frying Pan (defindex 1071, a genuinely distinct all-class item, not an
// attribute-driven reskin -- it's part of the general weapon list below,
// picked up automatically since it's used by all 9 classes), and Australium
// weapons.
//
// Australium items are a special case of the "Upgradeable ..." exclusion
// above: real Australium weapons use that SAME "Upgradeable"/paintkit-base
// defindex (quality Strange + attribute 2027 "is_australium_item" + style 1
// instead of a paintkit attribute), not the plain weapon's defindex --
// confirmed via the actual leaked Valve client source
// (ValveSoftware/source-sdk-2013, econ_item_view.cpp's
// CEconItemView::GetSkin): it only honors the style field at all if the
// item's OWN defindex declares a styles table in the schema, which only the
// "Upgradeable" defindexes do. So the 11 AUSTRALIUM_ITEMS entries below
// reuse those defindexes under a distinct "Australium <Weapon>" name (see
// gc_client_tf2.cpp / inventory.cpp for how the "Australium " prefix in the
// inventory file maps to these, alongside the "[effect]ItemName" bracket
// notation for Unusuals).
//
// "min_ilevel"/"max_ilevel" only appear where a weapon's real schema entry
// requires a level other than the default of 1 (e.g. Force-a-Nature needs
// exactly 10) -- CSOEconItem.level must fall within these bounds or the real
// client silently rejects the item and falls back to the class's default
// weapon for that slot instead of equipping it (see item_schema.h/
// inventory.cpp/gc_client_tf2.cpp for how this flows through to the wire).
